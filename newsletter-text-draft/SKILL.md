# 뉴스레터 텍스트 초안 스킬 (범용)

> **입력**: 카테고리 + 주제(brief) → **출력**: 이번 호 구조 레이아웃 + 슬롯 라벨이 달린 본문 텍스트
> 특정 브랜드·조직에 종속되지 않는다. HTML·디자인 렌더링은 만들지 않는다.

## 핵심 동작

1. 입력된 **카테고리**로 뉴스레터 **유형**을 판별한다.
2. 유형에 맞는 **섹션 레이아웃**을 구성한다 (어떤 섹션을 어떤 순서로 둘지).
3. 각 섹션의 **슬롯**에 주제(brief)를 풀어 카피를 채운다.
4. 출력 맨 앞에 **`## LAYOUT`** 블록으로 이번 호 구조를 먼저 보여준 뒤 본문을 잇는다.

---

## 작업 흐름 (항상 이 순서)

1. **입력 파악**
   - 필수: 카테고리(1개 이상) + 각 카테고리의 주제/내용(brief)
   - 선택: 발행일, 호 제목, 브랜드/발신자명, 목표 길이, 톤
   - 부족하면 **1~2개만** 짧게 되묻는다 (예: 발행일, 발신자명).
2. **references/section-structure.md** 읽기 — 섹션·슬롯 라이브러리와 유형별 레이아웃 패턴 확인
3. **references/copy-principles.md** 읽기 — 톤·길이·강조·금지 패턴 등 일반 카피 규칙 확인
4. **유형 판별 & 레이아웃 구성**
   - 입력 카테고리를 섹션-structure의 유형 표에 매핑한다.
   - 어디에도 안 맞으면 카테고리별로 섹션을 직접 조합한다 (1 카테고리 ≈ 1 섹션 블록).
5. **슬롯 채우기** — copy-principles 톤으로 각 슬롯에 카피 작성
6. **자가 점검** (아래 체크리스트) → 사용자에게 텍스트로 제시
   (요청 시 `Write` 도구로 `.md` 파일 저장)

---

## 출력 포맷 (슬롯 라벨 마크다운)

맨 앞에 **레이아웃 개요**를 두고, 이어서 각 섹션을 `##` 헤딩으로, 슬롯을 `슬롯명: 값` 형태로 출력한다.
값이 여러 문단이면 라벨 아래 줄에 그대로 적는다. 선택 슬롯은 내용이 있을 때만 출력한다.

```
# {HEADLINE}
> 발행일: {ISSUE_DATE} · 발신: {SENDER}

## LAYOUT (이번 호 구조)
유형: {뉴스레터 유형}
구성: HEADER → SECTION 0(오프닝) → SECTION 1(메인) → … → FOOTER

## HEADER
HEADLINE: ...
SUBLINE: ...

## SECTION 0 — 오프닝
TAG: ...
TITLE: ...
BODY:
(1~3문단)

## SECTION 1 — {카테고리명}
NUM: ①
TITLE: ...
BODY:
(2~3문단)
QUOTE: ▶ ...

## FOOTER
CTA: [버튼1 라벨] / [버튼2 라벨]
CONTACT: {연락/구독/링크 안내}
```

> 슬롯명·섹션명은 references/section-structure.md 의 정의를 사용한다.
> 라벨을 달면 사람이 읽기도 쉽고, 이후 HTML 변환 시 슬롯 매핑이 바로 된다.

---

## 자가 점검 체크리스트 (제시 전)

- [ ] 입력 카테고리가 모두 레이아웃에 반영되었는가 (누락된 주제 없음)
- [ ] 맨 앞 `## LAYOUT` 블록으로 이번 호 구조를 먼저 보여줬는가
- [ ] 각 섹션의 필수 슬롯이 빠짐없이 채워졌는가
- [ ] 전체 길이가 목표 범위(기본 **2,000~4,000자**, 공백 포함) 안인가
- [ ] 헤드라인은 질문형 남용 없이 **선언형·구체형** 인가
- [ ] 문단당 3~4문장 이내, 한 번에 나열은 최대 3항목인가
- [ ] 강조는 핵심어 1~2개만 (텍스트에서는 `**굵게**` 로 표기)
- [ ] 톤이 일관적인가 (입력 톤 또는 기본 톤 유지, 강의체·공지체 혼입 없음)
- [ ] 브랜드/제품명·수치·링크는 **입력에서 받은 값만** 사용했는가 (임의 창작·과장 없음)

---

## 빠른 참조 — 범용 카피 규칙

- 헤드라인: 질문형 남용 금지 → **선언형/동사형** 선호
- 과장 약속 금지: "모든 것을 해결", "단 N시간 만에" 류 비현실적 효율 프레임 지양
- 사과 과잉 대신 **업데이트 안내** 프레임 ("정정합니다" → "정확히 업데이트했습니다")
- 수치는 **단위와 함께** ("70% 단축"처럼), 출처/근거가 입력에 없으면 만들어 넣지 않는다
- 브랜드·제품·가격·연락처는 입력에 있는 것만 기입한다 (없으면 `{플레이스홀더}` 로 비워 둠)

> 이 스킬은 **텍스트만** 만든다. HTML·인라인 스타일·색상 토큰은 다루지 않는다.
> HTML 페이지가 필요하면 산출된 텍스트를 별도 HTML 생성 스킬에 입력한다.
