# 뉴스레터 섹션·슬롯 구조 (텍스트 초안용 · 범용)

> 이 파일은 텍스트 초안의 **구조(레이아웃 + 슬롯)** 만 정의한다.
> 디자인·HTML 렌더링은 이 스킬의 범위가 아니다.

---

## 전체 구조 개요

```
HEADER          — 발행일 · 타이틀 · 부제
SECTION 0       — 오프닝 (이번 호 핵심 메시지 1개)
SECTION 1..N    — 본문 블록 (입력 카테고리당 1블록, 유형에 맞는 슬롯 세트 사용)
FOOTER          — CTA + 연락/구독 안내
```

기본 목표 길이: **2,000~4,000자** (공백 포함). 입력에서 길이를 지정하면 그 값을 따른다.

---

## 유형별 레이아웃 패턴

입력된 카테고리 조합으로 가장 가까운 유형을 고르고, 그 레이아웃으로 섹션을 배치한다.
어디에도 맞지 않으면 **카테고리 1개 = 본문 블록 1개**로 직접 조합한다.

| 유형 | 대표 카테고리 | 레이아웃 |
|------|---------------|----------|
| 트렌드/인사이트 호 | 트렌드, 분석, 칼럼 | HEADER + S0 + S1(메인 토픽 1~2) + FOOTER |
| 큐레이션/링크 모음 호 | 추천, 읽을거리, 도구 모음 | HEADER + S0 + S1(큐레이션 리스트) + FOOTER |
| 행사/이벤트 안내 호 | 행사, 모집, 세미나 | HEADER + S0 + S2(이벤트 안내) + FOOTER |
| 제품·론칭 호 | 출시, 업데이트, 릴리스 | HEADER + S0 + S1(제품 소개) + S2(등록/안내) + FOOTER |
| 인터뷰/스토리 호 | 인터뷰, 사례, 후기 | HEADER + S0 + S1(스토리 본문) + FOOTER |
| 리캡/요약 호 | 주간 요약, 월간 결산 | HEADER + S0 + S1(하이라이트 2~3) + S2(다음 안내) + FOOTER |
| 종합 호 | 위 여러 카테고리 혼합 | HEADER + S0 + S1 + S2 + FOOTER |

> 본문 블록은 필요하면 여러 개로 반복한다 (SECTION 1, SECTION 2, …).
> 각 블록은 입력 카테고리의 성격에 따라 아래 **블록 유형**에서 슬롯 세트를 고른다.

---

## 공통 섹션 슬롯 (✅=필수)

### HEADER
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `ISSUE_DATE` | 선택 | 예: 2026년 6월 6일 토요일 (입력 없으면 생략 또는 `{발행일}`) |
| `SENDER` | 선택 | 발신자/브랜드명 (입력 값만) |
| `HEADLINE` | ✅ | 이번 호 주제를 담은 선언형·동사형 문장 |
| `SUBLINE` | 선택 | 이번 호 미리보기 한 줄 |

### LAYOUT (구조 개요 — 출력 맨 앞)
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `유형` | ✅ | 위 표에서 고른 뉴스레터 유형 |
| `구성` | ✅ | 섹션 흐름을 화살표로 표기 (HEADER → S0 → … → FOOTER) |

### SECTION 0 — 오프닝
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 상단 뱃지 텍스트 (예: Editor's Note, Core Message) |
| `TITLE` | ✅ | 오프닝 대제목 (2줄 이내) |
| `BODY` | ✅ | 본문 1~3문단. 이번 호 맥락·핵심 메시지 |

작성 원칙: 이번 호를 왜 읽어야 하는지 → 핵심 한 가지 → 본문으로 자연스럽게 연결.

### FOOTER
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `CTA` | ✅ | 주요 행동 유도 버튼 1~2개 라벨 |
| `CONTACT` | 선택 | 연락/구독/문의/구독해지 등 안내 (입력 값만) |

## 본문 블록 유형 (SECTION 1..N 에 골라 쓴다)

### A. 토픽/트렌드 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `NUM` | 선택 | "①", "Topic 1" 등 번호 |
| `TITLE` | ✅ | 읽고 싶어지는 훅이 담긴 대제목 |
| `BODY` | ✅ | 설명 2~3문단 (상황 → 의미 → 시사점) |
| `QUOTE` | 선택 | ▶ 핵심 인사이트 1문장 |
| `LINK` | 선택 | 더 읽기/출처 안내 (입력 값만) |

### B. 큐레이션 리스트 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 묶음 제목 (예: 이번 주 읽을거리) |
| `ITEMS` | ✅ | 항목 나열. 각 항목 `제목 — 한 줄 요약 (링크)` 형식, 3~6개 |

### C. 제품/업데이트 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 뱃지 (예: 새 기능, v2.0) |
| `TITLE` | ✅ | 제품/업데이트 제목 |
| `BODY` | ✅ | 무엇이 바뀌었고 왜 유용한지 |
| `STATS` | 선택 | 숫자 강조 1~3개 — `라벨=값` 나열 |
| `CTA` | 선택 | 사용/등록 유도 (입력 값만) |

### D. 이벤트 안내 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 파트 태그 (예: UPCOMING EVENT) |
| `TITLE` | ✅ | 행사명 |
| `DESC` | ✅ | 한 줄 설명 |
| `INFO_CHIPS` | ✅ | 일시 / 장소 / 모집인원 등 — 각 `라벨=값` |
| `TIMETABLE` | 선택 | 세션 시간표 (시간 / 내용 / 비고) |
| `NOTICE` | ✅ | 신청 방법·마감 안내 (선착순이면 명시) |

### E. 인터뷰/스토리 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 예: Interview, Case Study |
| `TITLE` | ✅ | 인물/사례 제목 |
| `BODY` | ✅ | 스토리 본문 (배경 → 전개 → 결과/배움) |
| `PULLQUOTE` | 선택 | “ 인상적인 인용 한 줄 ” |

### F. 소식/하이라이트 블록
| 슬롯 | 필수 | 설명 |
|------|------|------|
| `TAG` | 선택 | 뱃지 (예: 🎉 소식) |
| `TITLE` | ✅ | 소식 제목 |
| `BODY` | ✅ | 본문 (구체 맥락·수치는 입력 값만) |
| `STATS` | 선택 | 숫자 강조 1~3개 — `라벨=값` 나열 |

## 참고 예문 패턴 (범용)

- 오프닝: "이번 호는 [주제] 한 가지에 집중합니다. 핵심은 [한 문장]." 
- 토픽→시사점 전환: "그래서 무엇이 달라지는가 — [독자에게의 의미]."
- CTA 마무리: "자세한 내용은 [링크]에서 확인하세요." / "다음 호에서 [다음 주제]로 이어집니다."

> 브랜드·제품·가격·링크·연락처는 **입력에서 받은 값만** 채운다.
> 입력에 없는 고유명사/수치/약속은 만들어 넣지 않고 `{플레이스홀더}` 로 비워 둔다.
